DOCUMENTACIÓN DE ARQUITECTURA - LEARNATION OPS

1. ESPECIFICACIONES TÉCNICAS
--------------------------------
Backend:
- Lenguaje: Python 3.9+
- Framework: Flask (Modularizado con Blueprints)
- ORM: SQLAlchemy (Flask-SQLAlchemy)
- Migraciones: Flask-Migrate
- API: RESTful (JSON)

Frontend:
- Framework: React (Vite)
- Estilos: Tailwind CSS
- Iconografía: Lucide React

Base de Datos:
- Desarrollo Local: SQLite
- Producción: PostgreSQL

2. ESTRUCTURA DE ARCHIVOS ACTUAL
--------------------------------
/app
  /api                -> Blueprint principal de API JSON
    /auth.py          -> Endpoints de login/logout/me
    /admin.py         -> Endpoints de datos administrativos
  /admin              -> Lógica de Admin (ahora sirve datos vía API)
  /closer             -> Lógica de Closer
  /services           -> Capa de Lógica de Negocio (Business Logic)
  /models.py          -> Modelos de BD
  /__init__.py        -> Configuración (CORS, CSRF exemptions para API)

/frontend             -> Aplicación React SPA
  /src
    /components       -> Layouts y componentes reutilizables
    /pages            -> Vistas principales (Dashboard, Login)
    /services         -> Clientes API (Axios) y AuthService
    /App.jsx          -> Routing
    /main.jsx         -> Punto de entrada
  /package.json       -> Dependencias Node.js
  /vite.config.js     -> Configuración de compilación y proxy
  /tailwind.config.cjs -> Configuración de estilos

/migrations           -> Migraciones de BD
.gitignore            -> Excluye node_modules, env, instance, etc.
run.py                -> Servidor Flask
config.py             -> Variables de entorno

3. FLUJO DE TRABAJO
----------------------------------------
- Desarrollo Frontend: `cd frontend && npm run dev` (Vite proxy a :5000)
- Desarrollo Backend: `python run.py`
- Producción: Las dependencias se instalan en el servidor (`npm install` / `pip install`), NO se suben al repositorio.
